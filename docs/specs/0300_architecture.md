## Package Structure

```
FSharp.Windows.Dsl              вҶ’ virtual tree, reconciler, hot reload, props, VirtualTree queries
FSharp.Windows.Dsl.Elmish       вҶ’ Program.runWindow, Cmd extensions, ComponentContext, AppBus, memoization
FSharp.Windows.Dsl.Controls     вҶ’ codegen'd WPF control wrappers
FSharp.Windows.Dsl.Testing      вҶ’ TestProgram, CmdTest (for test projects)
FSharp.Windows.Dsl.Codegen      вҶ’ reflection-based code generation CLI tool
FSharp.DevExpress.Wpf           вҶ’ codegen'd DevExpress wrappers (mirrors vendor namespace)
FSharp.Telerik.Wpf              вҶ’ codegen'd Telerik wrappers (mirrors vendor namespace)
FSharp.Syncfusion.Wpf           вҶ’ codegen'd Syncfusion wrappers (mirrors vendor namespace)
```

`FSharp.Windows.Dsl` has NO Elmish dependency вҖ” the DSL layer is independent.
Third-party packages are optional. Community can generate wrappers for any WPF control library.

## Key Boundaries

| Layer | Responsibility | Never does |
|---|---|---|
| User Application | init/update/view, domain logic | Direct WPF manipulation |
| Elmish MVU Loop | Message dispatch, state management | UI rendering |
| DSL Layer | Virtual tree construction | Touching live WPF controls |
| Reconciler | Diff + patch live WPF tree | Business logic |
| WPF Managed | Layout, input, templating | вҖ” |
| milcore | GPU rendering | вҖ” |
